# Sales Shipments

Sales shipments are read-only records created when goods are shipped to customers. They are generated by posting sales orders or by the ship action. Shipments cannot be created or modified via the API.

---

## Base Endpoint

```
{{BC_BASE_URL}}/companies(name='{{BC_COMPANY_NAME}}')/salesShipments
```

---

## Operations

### 1. List All Sales Shipments

```http
GET {{BC_BASE_URL}}/companies(name='{{BC_COMPANY_NAME}}')/salesShipments
Authorization: Bearer {access_token}
Accept: application/json
```

**Response:**

### 2. Get a Single Shipment

```http
GET {{BC_BASE_URL}}/companies(name='{{BC_COMPANY_NAME}}')/salesShipments({id})
Authorization: Bearer {access_token}
Accept: application/json
```

### 3. Get Shipment Lines

```http
GET {{BC_BASE_URL}}/companies(name='{{BC_COMPANY_NAME}}')/salesShipments({ssId})/salesShipmentLines
Authorization: Bearer {access_token}
Accept: application/json
```

**Response:**

```json
{
  "value": [
    {
      "id": "b2c3d4e5-f6a7-8901-bcde-f23456789012",
      "documentId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
```

> This endpoint is **read-only** (GET only). Shipments are created by posting sales orders.

---

## OData Query Examples

### Filter by customer

```
$filter=customerNumber eq '10000'
```

### Filter by date range

```
$filter=shipmentDate ge 2024-06-01 and shipmentDate le 2024-06-30
```

### Filter by order number

```
$filter=orderNumber eq 'SO-1001'
```

## Common Errors

| Status | Error                  | Cause & Resolution                                                                 |
|--------|------------------------|------------------------------------------------------------------------------------|
| `401`  | Unauthorized           | Access token is missing, expired, or invalid.                                      |
| `404`  | Not Found              | The shipment does not exist. Verify the ID.                                        |
